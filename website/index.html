Sensors

What it monitors

BME680 and MAX9814 capture four environmental signals every 10 seconds and stream them to the cloud.

Temperature

Bosch BME680. Classrooms above 28 C can reduce cognitive performance by up to 20%.

Ideal: 18 to 28 C

Humidity

Capacitive sensing via BME680. Low humidity spreads viruses; high humidity promotes mould.

Ideal: 20 to 60%

VOC Air Quality

Metal-oxide gas sensor measures volatile organic compounds. Calibrated to an IAQ index by the Bosch BSEC2 library.

Good: IAQ below 50

Noise Level

Analog envelope from MAX9814 auto-gain mic, sampled by the ESP32 ADC. Classified as Quiet, Moderate, or Loud.

Quiet: below 1000 ADC
Architecture

How it works

Three layers: firmware reads and classifies, Supabase stores, and the React Native app delivers it in real time.

1

Firmware on ESP32

Reads the BME680 via I2C using the Bosch BSEC2 library and the MAX9814 via ADC. Every 10 seconds it runs CelsiusIntelligence on-device inference and POSTs a JSON payload to Supabase over HTTPS. WiFiManager stores credentials in flash -- no hardcoding, no reflashing for network changes.

C/C++ Arduino BSEC2 TFLite Micro WiFiManager
2

Supabase Backend

PostgreSQL hosted on Supabase. The ESP32 inserts rows via REST API. Each record includes temperature, humidity, air quality, noise, IAQ score, and the CelsiusIntelligence classification label. No custom server -- Supabase handles auth, storage, and realtime.

Supabase PostgreSQL REST API
3

React Native App

Expo app with three tabs -- Dashboard, Analytics, History. Polls every 5 seconds. Runs 2-sigma anomaly detection on the last 50 readings. Push notifications fire on warning or poor status transitions. Claude Haiku provides contextual teacher tips.

React Native Expo SDK 54 TypeScript
Intelligence

CelsiusIntelligence

On-device TinyML runs on the ESP32 itself. No cloud inference, no WiFi dependency. A second statistical system in the app catches anomalies the model might miss.

On-Device TinyML

A three-layer MLP trained on labelled classroom readings using TensorFlow, quantized to int8, and deployed via TFLite Micro. CelsiusIntelligence classifies every reading in microseconds on the ESP32 itself. Trained by running train_celsiusintelligence.py on the live Supabase dataset.

Statistical Anomaly Detection

The app fetches the last 50 readings on startup and computes a rolling baseline per sensor channel. Any reading more than two standard deviations from the mean triggers an anomaly flag -- no labelled data required. It learns the normal range of your specific room.

mu = mean(last 50 readings)
sigma = std(last 50 readings)
anomaly = |x - mu| > 2 * sigma

Baseline refreshes every 5 minutes. Runs independently per sensor channel.

Alerts

Three-tier alert system

Teachers do not need to watch a screen. Hardware and app signal status independently. Push notifications fire the moment conditions change.

Good

Green LED / Silent

All sensors within range. Temperature 18 to 28 C, humidity 15 to 65%, VOC above 45 kOhm. No action needed.

Warning

Yellow LED / One beep / Push notification

Air quality moderate, or two or more sensors at warning level. App sends a push notification and the AI tip updates with a relevant action.

Poor

Red LED / Triple buzzer / Push notification

Any sensor in the danger zone. Buzzer sounds three bursts. App fires an alert dialog and sends an immediate push notification. Open windows now.
